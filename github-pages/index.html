Install / Update

Bootstrap or refresh a root

Run the stable update script from the directory where you want the flget root to live. 

cd C:\tools\flget
powershell -c "irm https://flatina.github.io/flget/update.ps1 -OutFile .\update.ps1;.\update.ps1"

Self-update later with flget update or .\update.ps1. Run .\REGISTER_PATH.ps1 only if you explicitly want PATH registration.

Quick Start

Use the root immediately

.\activate.ps1
flget --version

flget install 7zip --source scoop
flget install ripgrep --source ghr
flget install typescript --source npm
flget install pnpm --source npmgh
flget skills install openai/codex

Use a fully-qualified ref such as ghr:owner/repo when you need an exact non-interactive install.

Sources

Standalone packages and agent skills

flget is not a drop-in replacement for scoop or npm. It focuses on portable, directory-isolated roots and source-scoped installs.

scoop

Install Windows apps from Scoop manifests while keeping the flget root isolated.

npm

Install npm registry packages into the root without using global -g.

ghr

Resolve and install packages directly from GitHub Releases.

npmgh

Install npm-style packages from GitHub source repositories.

skills

Store skill installs under agents/skills/ and expose declared shims.

env refresh

flget env refreshes env cache files and ensures static root scripts exist.

Offline Roots

Reuse an existing root as a source

In air-gapped environments, copying the entire root directory is usually enough. If you only need a fresh shell plus local package reuse, copy the root files and add an existing root as a source. 

mkdir C:\offline-flget
copy K:\flget\*.* C:\offline-flget   # root files only, not recursive

cd C:\offline-flget
flget root add K:\flget
flget install 7zip --source scoop

Secrets

Plain .env or encrypted .flenc

Secret lookup uses process env first, then root and shared secret files. Encrypted .flenc files require FLGET_SECRETS_KEY.

Lookup order: process env -> .env -> .secrets/.env -> .secrets/<profile>.env -> encrypted .flenc variants.

If you need stronger sharing, rotation, or auditability, inject secrets from an external tool before running flget.

Assets

Published root files

The update flow publishes the static PowerShell entrypoints plus the runtime bundle used to create or refresh a flget root.

update.ps1

Stable manual install and update entrypoint.

 activate.ps1

Session-level environment entrypoint that recreates missing shims.

 REGISTER_PATH.ps1

Optional PATH registration script. Not required for core flows.

 bootstrap.ps1

Compatibility wrapper that delegates to update.ps1.

 dist/flget.js

Published CLI bundle used by the root update flow.

 README

Full command reference, root layout, and troubleshooting notes.
